How It Works

Each framework is isolated in its own folder and can run independently. UI automation uses Selenium or Playwright, API coverage uses RestAssured or Playwright request contexts, and GitHub Actions publishes the latest generated reports.

Retry and Artifact Flow

The Playwright example includes a dedicated rerun test that intentionally fails once and passes on retry. This makes the report show the exact artifacts engineers usually need during triage: failure screenshot, retained video, and first-retry trace.

To find it in the published Allure report, open the Playwright report and go to Suites. Then open demo-rerun.spec.ts, select the Demo retry flow suite, and open fails once for artifacts and passes on rerun.

Inside that test result, review the attachments from the failed first attempt to see the screenshot, retained video, and retry trace.
